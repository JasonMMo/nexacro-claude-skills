## What shipped

### 1. Plugin #3 — `nexacro-fullstack-starter` (v0.1.0)

A Claude Code plugin that scaffolds a full-stack Nexacro N v24 project by sparse-cloning one of 8 runners from the companion monorepo. Parameters:

- `JDK` ∈ { `8`, `17` }
- `FRAMEWORK` ∈ { `spring-boot`, `spring-mvc`, `egov-boot`, `egov-mvc`, `webflux` }
- `PROJECT_NAME`, `BACKEND_URL`, `CONTEXT_PATH`, `SERVER_PORT`

Rules are enforced in `assets/matrix.json`:

- `servletApi = jdk >= 17 ? jakarta : javax`
- `springMajor = jakarta ? 6 : 5`
- `bootMajor = jakarta ? 3 : 2`
- **Rejected:** `egov-mvc + jdk17`, `webflux + jdk8`

Plugin passes structural and dry-run install verification (see `scripts/verify_plugin_structure.py`, `scripts/dryrun_plugin_install.py`). Marketplace now advertises **3** plugins (was 2).

### 2. Companion monorepo — `JasonMMo/nexacroN-fullstack`

New public GitHub repo containing:

- `samples/shared-business/{jdk17-jakarta, jdk8-javax}/` — shared Nexacro envelope + MyBatis business logic for both servlet lanes
- `samples/runners/boot-jdk17-jakarta/` — **proven** (Spring Boot 3.3, Spring 6, JDK 17, Jakarta)
- `samples/runners/boot-jdk8-javax/` — **proven** (Spring Boot 2.7.18, Spring 5, JDK 8 target, Javax)
- `samples/runners/{mvc-*, egov*, webflux-*}/` — **scaffolded** (README-only; business tree will slot in during v1.8.x point releases)
- `samples/seed-data/{schema.sql, data.sql}` — HSQL 2.7.3 in-memory seed (`^^` separator; PSM WHILE loop generates 1000 rows of `LARGE_DATA`)

Monorepo tags shipped: `v0.1.0-skeleton`, `v0.2.0-flagship`, `v0.3.0-dual-lane`.

### 3. Dual-lane axiom — proven

The architectural bet was that two lanes (`jakarta` vs `javax`) diverge only on:

- servlet package (`jakarta.servlet.*` vs `javax.servlet.*`)
- maven-compiler `<release>17>` vs `<source>1.8><target>1.8>`
- Spring/Boot major versions
- `mybatis-spring-boot-starter` major version (3.x vs 2.3.2 pin)

…and the rest (Nexacro envelope, business services, MyBatis XML, HSQL schema) is **literally shared source**.

Both lanes boot end-to-end against the same seed DB and pass the same three smoke tests:

| Test | jakarta (8080) | javax (8081) |
|---|---|---|
| `POST /uiadapter/login` | PASS | PASS |
| `POST /uiadapter/board/select` (5 rows) | PASS | PASS |
| `POST /uiadapter/large/page` (10 rows, total=1000) | PASS | PASS |

## Known gaps — follow-up plans

The 6 non-proven runners (`mvc-jdk17-jakarta`, `mvc-jdk8-javax`, `egov5-boot-jdk17-jakarta`, `egov4-boot-jdk8-javax`, `egov4-mvc-jdk8-javax`, `webflux-jdk17-jakarta`) ship as **scaffold-only** at v1.8.0. Each one is a repeat of the Plan 2 / Plan 3 pattern against the same business trees and will land in a v1.8.x point release without requiring a new minor.

Other deferred items:

- **hsqldb 2.7.3 runtime**: javax lane's compile target is JDK 8 but the embedded HSQL driver needs JDK 11+ at runtime. Documented in the runner pom. A future hardening pass could add a Maven toolchain declaration.
- **Phase 3 archive buttons**: banner template is ready; the physical "Archive project" click on each of the 8 GitLab repos happens on **2026-05-15** after a 2–3 week observation window.
