## What shipped

### 1. Monorepo `core/` — first-party nexacro modules (partial, honest)

`nexacroN-fullstack` now carries a `core/` directory with the first-party nexacro integration artefacts that were missing from v1.8.1:

```
core/
├── libs/
│   └── jakarta/
│       ├── nexacroN-xapi-jakarta-1.2.4-20260312.005603-1.jar   (520 KB)
│       └── nexacroN-xeni-jakarta-1.5.21-20260312.012638-1.jar  (202 KB)
│
├── uiadapter-javax/                  — 3 Maven submodules, 102 .java files
│   ├── uiadapter-spring-core         — 65 .java (MVC surface)
│   ├── uiadapter-spring-dataaccess   — 25 .java (ibatis / jdbc / mybatis)
│   └── uiadapter-spring-excel        —  2 .java (Excel servlet)
│
└── README.md
```

**Provenance:**
- `libs/jakarta/*.jar` — copied verbatim from `asis/jakarta-example` build tree (tobesoft-internal Nexus SNAPSHOT timestamp `20260312`), identical to the binaries running the existing production jakarta sample.
- `uiadapter-javax/` — verbatim copy of `nexacro-webflux/asis/uiadapter/uiadapter-spring/` (Spring 5 / javax lane reference). IDE metadata and javadoc output stripped; Maven source tree intact.

Monorepo tag: **`v0.5.0-core`**.

### 2. Runner pom wiring

**`boot-jdk17-jakarta/pom.xml`** now declares optional `systemPath` dependencies on the jakarta xapi + xeni jars:

```xml
<dependency>
  <groupId>com.nexacro</groupId>
  <artifactId>nexacroN-xapi-jakarta</artifactId>
  <version>1.2.4-SNAPSHOT</version>
  <scope>system</scope>
  <systemPath>${project.basedir}/../../../core/libs/jakarta/nexacroN-xapi-jakarta-1.2.4-20260312.005603-1.jar</systemPath>
  <optional>true</optional>
</dependency>
```

`<optional>true</optional>` keeps the runner compiling against its inline `com.nexacro.fullstack.business.xapi.*` / `…uiadapter.*` classes by default. Consumers who want the first-party binaries flip the scope to `compile` and delete the inline copies. Build-verified: `mvn compile` → BUILD SUCCESS.

**`boot-jdk8-javax/pom.xml`** gains a documentation block pointing to `core/uiadapter-javax/` as reference source and documenting the remaining xapi/xeni (javax) binary gap. No dependency changes. Build-verified.

### 3. Deferred asymmetries documented — not hidden

| Missing piece | Status | Restore path |
|---|---|---|
| `core/libs/javax/nexacroN-xapi-javax-*.jar` | not in this workspace | tobesoft Nexus (`mangosteen.tobesoft.co.kr`) with credentials |
| `core/libs/javax/nexacroN-xeni-javax-*.jar` | not in this workspace | tobesoft Nexus |
| `core/uiadapter-jakarta/` (source) | the jakarta runner loads uiadapter as a pre-built jar from tobesoft Nexus; source not in workspace | tobesoft Nexus, or mechanical port of `uiadapter-javax/` (`javax.servlet.*` → `jakarta.servlet.*` + small Spring 5 → 6 SPI shifts) |

The asymmetry is intentional: both proven runners already ship the nexacro envelope surface **inline** inside their `shared-business-*` trees, so the missing `core/` pieces do not block compile or boot. `core/` is provided so consumers can **replace** the inline surface with the official first-party modules when they want to.

Full reference: `core/README.md` in the monorepo.

## Deferred to v1.8.3+

The following items were originally queued for v1.8.2 but are deliberately deferred:

### 14-endpoint controller contract alignment

Current runner controllers cover ~8 endpoints. The spec's 14-endpoint contract (`/select_data_single.do`, `/advancedUploadFiles.do`, `/multiDownloadFiles.do`, `/streamingVideo.do`, `/select_testDataTypeList.do`, `/update_deptlist_map.do`, `/sampleLargeData.do`, `/search_manyColumn_data.do`, `/relay/exim_exchange.do`, etc.) requires backing service + DAO + MyBatis XML + HSQL seed extensions that exceed the v1.8.2 envelope. Tracked for v1.8.3.

### 6 non-proven runners (`mvc-*`, `egov*`, `webflux-*`)

Remain scaffold-only (unchanged from v1.8.0). Business tree port still pending.

### javax xapi/xeni + jakarta uiadapter source

Blocked on tobesoft Nexus access from this workspace. Will land when the real artefacts are available from a reachable repository.

## How to use

No consumer-facing command change from v1.8.1:

```
/plugin marketplace update nexacro-claude-skills
/nexacro-fullstack-starter
```

After scaffold, the generated project's `samples/runners/boot-jdk17-jakarta/pom.xml` already carries the optional first-party nexacro dependencies. To opt in, flip `<scope>system</scope>` → `<scope>compile</scope>` on the two entries and delete the inline `com.nexacro.fullstack.business.xapi.*` / `uiadapter.*` classes under `shared-business-jdk17-jakarta`. See `core/README.md` for the full migration recipe.
