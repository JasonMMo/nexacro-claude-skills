## What shipped

### 1. `nxui/packageN` — full Nexacro N v24 project body

The `nexacroN-fullstack` monorepo now ships the complete `packageN` project tree under `nxui/packageN/`, sourced from `asis/jakarta-example/src/main/nxui/packageN` (the same tree the existing jakarta sample runner uses in production).

- **56 MB, 1531 files**
- `packageN.xprj` / `packageN.xadl`
- `typedefinition.xml` — **Services block patched** for scaffold-time token substitution
- `_resource_/` — theme / initvalue / images / font / xcss
- `nexacrolib/` (1379 files, 55 MB) — runtime components, framework, resources
- `frame/` (8 frames) — frameBottom / frameLeft / frameLogin / frameMain / frameMDI / frameTop / frameWork / frameWorkTitle
- `pattern/` (4 patterns) — transaction, excel, FileUpTransfer, largeData
- `sample/` (13 forms) — Calendar, Grid, Message, Popup, Script, FileUpDownloadTrans, streaming video, bulk columns, etc.
- `cmm/` — cmmAlert, cmmConfirm popups
- `lib/cmmInclude.xjs`, `debugging/`, `images/`

Monorepo tag: **`v0.4.0-nxui-packageN`**.

#### Services contract — two-tier layout

The `typedefinition.xml` Services block is now restructured into:

**Core 9 static services** (canonical nexacroN v24 layout — never substituted):
- `theme`, `initvalue`, `imagerc`, `font` (resource)
- `extPrototype`, `lib` (js)
- `frame` (form)
- `xcssrc` (resource)
- `images` (file)

**Scaffold-time auxiliaries** (form-folder mounts + tokenised backend):
- `svcurl` → `{{BACKEND_URL}}` (replaced by plugin at scaffold)
- `cmm`, `sample`, `pattern` (form folders consumed by bundled navigation / popups)

Full reference: `nxui/README.md` in the monorepo.

### 2. Plugin `SKILL.md` — Step 6 nexacrodeploy build handoff

The plugin skill now inserts a dedicated `Step 6 — nexacro 빌드 / Nexacro build` between post-processing and final guidance:

- Runner-family → build-output path mapping (`boot-*` / `webflux-*` → `src/main/resources/static/packageN/`; `mvc-*` / `egov*-mvc-*` → `src/main/webapp/packageN/`)
- Copy-paste `nexacrodeploy.exe` command with the right flags
- Auto-invocation of the user-local `/nexacro-build` skill when present (`Skill(skill: "nexacro-build", args: "...")`)
- Graceful fallback to printed guidance when `/nexacro-build` is not installed
- Windows-only caveat documented for macOS / Linux users

The previous Step 6 (Final guidance) is now Step 7 and surfaces the build path + pointer to Step 6 in the next-steps list.

### 3. `.gitignore` — nexacro IDE cache

Monorepo `.gitignore` now ignores `$Geninfo$.geninfo` and `*.geninfo` (nexacro IDE local caches).

## Deferred to v1.8.2

The following items were in v1.8.1 scope at the start of the cycle but are deliberately deferred:

### `core/` module import — not functionally required

Investigation showed the two proven runners (`boot-jdk17-jakarta`, `boot-jdk8-javax`) already ship the nexacro envelope / uiadapter surface **inline inside their `shared-business-*` trees** (`com.nexacro.fullstack.business.xapi.*`, `com.nexacro.fullstack.business.uiadapter.*`). They do not load external `nexacroN-xapi-*.jar` / `nexacroN-xeni-*.jar` / `uiadapter-spring-*.jar` artefacts.

Importing the full xapi/xeni/uiadapter modules under `core/` would therefore be a **source-of-record swap**, not a fix — and the javax-lane binaries are not present in this workspace (they live on the tobesoft Nexus). The swap belongs in its own minor and will land when we wire the real artefacts for both lanes from a reachable repository.

### 14-endpoint contract alignment

Current runner controllers cover ~8 endpoints (`/login.do`, `/board/*`, `/dept/list.do`, `/dept/tree.do`, `/file/upload.do`, `/file/list.do`, `/file/download.do`, `/large/page`). The spec's 14-endpoint contract (`/select_data_single.do`, `/advancedUploadFiles.do`, `/multiDownloadFiles.do`, `/streamingVideo.do`, `/select_testDataTypeList.do`, `/update_deptlist_map.do`, `/sampleLargeData.do`, `/search_manyColumn_data.do`, `/relay/exim_exchange.do`, etc.) requires backing service + DAO + MyBatis XML + HSQL seed extensions that exceed the v1.8.1 patch-release envelope. Tracked for v1.8.2.

### 6 non-proven runners (`mvc-*`, `egov*`, `webflux-*`)

Ship as scaffold-only at v1.8.1 (unchanged from v1.8.0). Business tree port lands in a later v1.8.x.
